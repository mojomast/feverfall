# Audio Contracts

Checkpoint 5 routes the `FeedbackEvent` presentation stream into an explicit runtime audio model without mutating deterministic gameplay state.

## Runtime buses

The five required buses are modeled as runtime channels in `RUNTIME_BUSES`:

- `CollisionPlinks` — peg and orange plinks, rate-limited for dense peg hits.
- `ComboPercussion` — combo threshold percussion accents.
- `RewardStingers` — reward, relic, fever, bucket, and positive stingers.
- `AmbientBoardHum` — low board hum, tension risers, and the fever silence gate.
- `UiConfirmations` — launch, loss/near-miss downbeats, and UI confirms.

Each bus has a stable channel name, volume, and minimum interval. The Bevy feature enables built-in audio/wav support; bus semantics remain explicit in the FeverFall adapter so Kira or a mixer can replace the backend later.

## Feedback mapping and safeguards

Every `FeedbackKind` maps to one or more `AudioCue`s. `ExtremeFever` intentionally emits `SilenceGate` before `FeverSting` so the victory sting is not stacked over a tension riser. Loss and near-miss map only to restrained downbeats and do not use victory layers.

Accessibility and ethics rules are preserved in the audio state:

- `mute_high_frequency_layers` drops plinks, shimmers, bells, and risers marked high-frequency.
- Requested intensity is clamped per event kind; loss and near-miss remain low intensity.
- More than 12 peg-hit samples collapse into a lower-frequency `ChordCluster` to avoid harsh high-rate plinks.

## Asset provenance

Runtime assets live under `game/assets/audio/` and are documented in `game/assets/audio/ATTRIBUTION.md`. Current C5 assets are CC0 fallback synthesis WAVs generated by FeverFall contributors; no unclear third-party audio is present.

## Validation

Required tests in `game/src/plugins/audio/mod.rs` cover:

- `all_feedback_kinds_have_audio_mapping`
- `all_audio_buses_have_runtime_channel`
- `loss_and_near_miss_do_not_use_victory_layers`
- `high_frequency_mute_omits_plinks`
- `extreme_fever_silence_gate_precedes_sting`
- `audio_asset_manifest_paths_exist`
